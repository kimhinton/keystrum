# Capacitor Setup — Bootstrap Guide

## Prerequisites

- Node 20+ installed
- For local Android builds: Android Studio + JDK 17 + ANDROID_HOME set
- For local iOS builds: macOS + Xcode 15+ (otherwise use GitHub Actions)
- Apple Developer Program ($99/yr) for iOS distribution
- Google Play Console ($25 one-time) for Android distribution

## One-time setup (5 min)

```bash
# 1. Install deps (Capacitor packages were added to package.json)
npm install

# 2. Build the static Next.js export
npm run build

# 3. Add native platforms (creates android/ and ios/ directories)
npx cap add android
npx cap add ios        # works on Windows to create files, but BUILD requires macOS

# 4. Sync web assets + plugins into native projects
npx cap sync
```

All four steps are combined in `bash scripts/setup-capacitor.sh`.

## Project structure after setup

```
keystrum/
├── capacitor.config.ts       # Capacitor config (already committed)
├── android/                   # Android Studio project (generated by cap add)
├── ios/                       # Xcode project (generated by cap add)
├── out/                       # Next.js static export (generated by npm run build)
└── .github/workflows/
    ├── android-build.yml     # Ubuntu runner → AAB artifact
    └── ios-build.yml         # macOS runner → xcarchive artifact
```

**Commit `android/` and `ios/` to git.** These directories contain the native shell
(Java/Kotlin for Android, Swift/Objective-C for iOS) and should be version-controlled.

Exception: don't commit per-developer files like `android/local.properties`,
`ios/App/Pods/`, and `ios/DerivedData/`. Add to .gitignore if not already.

## Development loop

### Android
```bash
npm run cap:sync            # build + sync (runs next build && npx cap sync)
npm run cap:android         # opens in Android Studio
# OR
npm run android:build-aab   # release AAB at android/app/build/outputs/bundle/release/
npm run android:build-apk   # release APK at android/app/build/outputs/apk/release/
```

### iOS (macOS only for local)
```bash
npm run cap:sync
npm run cap:ios             # opens in Xcode — cmd+R to run on simulator/device
```

### iOS (Windows → GitHub Actions)
```bash
git tag v0.1.0
git push origin v0.1.0     # triggers the iOS Build workflow
# OR manually: Actions tab → "iOS Build" → Run workflow
```

Artifact will be an unsigned `.xcarchive`. For real distribution, set up signing
(see note block at bottom of `.github/workflows/ios-build.yml`).

## Capacitor ↔ Next.js integration notes

### Service Worker (sw.js)
Capacitor WebView serves from `capacitor://localhost` (iOS) or `https://localhost`
(Android). Service Worker MAY fail to register on iOS. Current code swallows
registration errors silently — no action needed.

### PWA manifest (manifest.ts)
Ignored inside Capacitor (manifest is a PWA construct, Capacitor has its own
native manifests). Harmless to keep.

### Routing
Next.js static export produces full HTML pages per route. Capacitor's WebView
loads `index.html` at startup, and client-side routing takes over. All existing
`<Link>` navigation works unchanged.

### Audio / Wake Lock
- `MediaRecorder`: supported in WKWebView 14.3+ and Android WebView 90+. The mp4
  fallback in `packages/synth/src/guitar-synth.ts` is critical for iOS.
- `WakeLock API`: NOT supported in iOS WKWebView as of iOS 17. Screen dims during
  recording on native iOS app; user can disable iOS auto-lock in Settings as a
  workaround. Android WebView supports it.

### Native Share
`navigator.share()` works in Capacitor via the `@capacitor/share` plugin shim.
The existing RecordBar code uses `navigator.share()` directly — Capacitor bridges
this to native share sheet automatically.

## Splash screen (optional polish)

For native launch screens:

1. Install: `npm i @capacitor/splash-screen`
2. Generate assets: `npm i -D @capacitor/assets && npx capacitor-assets generate`
   - Requires `resources/icon.png` (1024×1024) and `resources/splash.png` (2732×2732)
3. Capacitor auto-wires splash on next `cap sync`

Not required for MVP submission — skip if shipping fast.

## Gotchas

- **Don't modify `public/sw.js` to hit `capacitor://localhost`** — it won't help,
  just let SW fail to register silently in native builds.
- **Google Play requires target SDK 34+** as of 2026. Check
  `android/app/build.gradle` → `targetSdkVersion = 34` after `cap add android`.
- **iOS 17 requires privacy manifest** (`PrivacyInfo.xcprivacy`) for apps that
  use certain APIs. keystrum uses MediaRecorder + local storage only — no
  manifest entries required, but the file must exist. Capacitor 6+ generates a
  minimal one automatically during `cap add ios`.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `Error: webDir 'out' not found` | Run `npm run build` before `cap sync` |
| `android/ missing` in CI | Run `npx cap add android` locally once, commit the dir |
| iOS build "No code signing identity" | Enable signing (see ios-build.yml notes) |
| White screen on Android launch | Check `android/app/src/main/assets/public/` has files — re-run `cap sync` |
| Audio silent on iOS launch | Web Audio needs user gesture — existing "press any key" is correct |
